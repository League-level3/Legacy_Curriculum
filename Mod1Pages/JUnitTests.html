JUnit Tests

When you have written a new program, you need to test if it is working properly. Also, if you need to make changes to an existing program, you need to make sure you haven't broken anything that worked before.

When we develop new code, we sometimes add print statements to check what is happening, but after we have done that we need to remove the print statements, and that changes the program. So how can we test a program without changing it?

JUnit provides us with a framework to test our code "from the outside". Overview of JUnit

For example, if we write a static method in an Algorithms class that is designed to add 2 numbers and return the sum, a JUnit test can be written to call that method, pass it 2 numbers and check the result against the expected answer. This test is not part of the Algorithms class, but an external testing class, and might appear as follows:

@Test
public void testAdd() {
     assertEquals(4, Algorithms.add(3,1));
}

We can use a variety of assertions that allow the program to test for different conditions. In the above example, the code is testing for the "equals" condition.

If a programmer later changes the Algorithms class for any reason, we can re-run this test to see if the add() method still works correctly.
